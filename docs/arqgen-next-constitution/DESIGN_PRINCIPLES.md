# Princípios de Design

## O design serve à decisão

O Arqgen Next deve ser elegante, calmo, mínimo, rápido e profissional. Esses atributos não são um estilo autônomo: existem para aumentar legibilidade, confiança e controle.

## Clareza antes de magia

Uma transição impressionante não compensa uma premissa oculta. Resultados gerados devem aparecer com origem, critérios e mudança. O momento de encanto é compreender algo difícil mais cedo.

## Confiança antes de estética

O design deve distinguir visualmente fato, premissa, inferência, recomendação e risco. Estados não podem depender apenas de cor. Números precisam de unidade, período, precisão adequada e acesso à fórmula.

## Densidade com controle

Este é um produto técnico. Não devemos eliminar informação importante para parecer simples. Devemos criar hierarquia:

- resumo para leitura executiva;
- comparação para decisão;
- detalhe sob demanda;
- evidência ao alcance de um gesto.

Espaço vazio é útil quando organiza atenção; prejudicial quando empurra conteúdo crítico para fora da vista.

## Canvas como pensamento

O canvas é o objeto dominante. Painéis flutuam ou se recolhem quando não são necessários. Seleções no comparador devem refletir no espaço; alertas espaciais devem apontar a regra ou premissa correspondente.

3D é usado quando esclarece altura, relação volumétrica ou insolação, não como decoração.

## Comparação explícita

Comparar é mais importante que admirar uma alternativa isolada. Preferir:

- colunas alinhadas;
- escalas consistentes;
- deltas absolutos e percentuais;
- destaques de diferença;
- filtros por objetivo;
- explicações de causa.

Evitar cards incomparáveis, rankings sem critérios e verde/vermelho quando “mais” não significa sempre “melhor”.

## Dois níveis de leitura

Para investigação técnica: alta precisão, fontes, fórmulas, geometria e histórico.

Para decisão executiva: objetivo, opções, trade-off dominante, risco, confiança e próximo passo.

Ambos devem derivar do mesmo modelo; não criar uma narrativa executiva desconectada da análise.

## Movimento

Animação mostra relação causal, atualização e mudança de estado. Deve ser curta, interrompível e respeitar redução de movimento. Nunca usar movimento contínuo para fingir que a IA está trabalhando.

## Escrita na interface

Curta, específica e orientada ao projeto.

Preferir “CA utilizado subiu para 3,72” a “Alteração aplicada com sucesso”. Preferir “Regra de vagas pendente” a “Atenção: há informações faltando”.

## Anti-padrões

- decoração futurista para sinalizar IA;
- gradientes e brilho sem função semântica;
- dashboards de KPIs sem decisão;
- chat ocupando mais atenção que o projeto;
- excesso de badges;
- precisão decimal falsa;
- alertas universais sem localização;
- controles repetidos em várias superfícies.

## Checklist

- A hierarquia começa pela decisão?
- O canvas permanece legível?
- Estados críticos funcionam sem cor?
- Comparações usam bases equivalentes?
- A fonte de uma conclusão é acessível?
- O modo executivo preserva ressalvas?
- Cada elemento visual tem função?
